Теперь объедини backend и frontend TransLogix в единый локально запускаемый проект.

Нужно:
1. Создать docker-compose.yml
2. Подключить:
   - frontend
   - backend
   - postgres
3. Настроить переменные окружения.
4. Обеспечить корректное сетевое взаимодействие контейнеров.
5. Добавить инициализацию БД и seed-данных.
6. Сделать так, чтобы проект запускался одной командой.

Требования:
- docker compose up --build должен поднимать систему без ручных правок
- frontend должен обращаться к backend по корректному URL
- backend должен ждать PostgreSQL или корректно переживать его поздний старт
- добавь README.md с пошаговой инструкцией

README должен содержать:
1. описание проекта
2. стек
3. дерево каталогов
4. команды запуска
5. demo credentials / role access
6. список реализованных фич
7. ограничения прототипа
8. идеи для развития

После генерации:
- проверь все команды запуска;
- проверь зависимости между сервисами;
- покажи итоговую структуру репозитория;
- перечисли, что ещё стоит вручную протестировать перед сдачей лабораторной.

Перед тем как генерировать код:
- проверь согласованность имен сущностей и API;
- не создавай лишнюю сложность;
- ориентируйся на работающий MVP;
- если есть спор между “красиво архитектурно” и “реально запускается для лабораторной”, выбирай второе;
- в конце каждой генерации делай self-check:
  1. broken imports
  2. missing env variables
  3. mismatch between frontend and backend contracts
  4. incomplete mock data
  5. run instructions completeness
