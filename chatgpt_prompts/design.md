Design a modern B2B SaaS web application interface for a logistics intelligence platform called TransLogix.

Product context:
TransLogix is an intelligent logistics system for delivery route optimization and warehouse inventory management. The platform is used by dispatchers, warehouse managers, operations managers, and couriers. It should look like a realistic enterprise logistics dashboard, not a generic startup landing page.

Design goals:
- clean, professional, operations-focused UI
- modern SaaS dashboard aesthetic
- high information density, but still readable
- suitable for desktop first, with responsive tablet layout
- dark text on light background
- strong visual hierarchy
- map + tables + KPI cards + alerts + status badges
- enterprise logistics feel similar to supply chain / fleet / warehouse software

Style direction:
- minimal and polished
- subtle shadows
- rounded cards
- neutral palette with blue as primary accent
- success/warning/error states clearly visible
- emphasis on operational clarity and quick scanning
- not playful, not overly colorful

Create screens for:
1. Login / role selector
2. Main dashboard
3. Orders management page
4. Routes management page
5. Warehouses and inventory page
6. Courier route view
7. Notifications / alerts page

Required layout patterns:
- left sidebar navigation
- top header with page title, search, user menu
- KPI cards row
- chart widgets
- table-heavy content areas
- right-side detail drawer or modal for selected records
- status badges for order and delivery states
- route visualization panel / map block
- alert panels for low stock and delayed deliveries

Main dashboard should include:
- total orders
- on-time delivery rate
- active routes
- delayed deliveries
- low stock alerts
- average ETA
- route efficiency trend chart
- delivery status breakdown chart
- active routes table
- urgent alerts section

Orders page should include:
- searchable/filterable table
- columns: order ID, customer, destination, priority, status, ETA, assigned route
- order detail panel
- action to assign to route

Routes page should include:
- list of routes
- route status
- number of stops
- assigned courier
- estimated completion time
- route detail panel with ordered stops
- recommendation widget “AI suggested route”

Inventory page should include:
- warehouse selector
- stock table
- low stock highlighting
- replenishment recommendation widget
- inventory metrics cards

Courier view should include:
- simple mobile-like layout inside the web app
- current route
- stop list
- delivery status actions
- compact KPI summary

Use realistic enterprise labels and sample data. 
Avoid marketing sections, hero blocks, testimonials, or pricing cards.
Focus entirely on application UI.
