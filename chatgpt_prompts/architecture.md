Проект: TransLogix.

Краткая идея системы:
TransLogix — интеллектуальная логистическая информационная система, оптимизирующая маршруты доставки и управление складскими запасами с помощью ИИ. Система должна поддерживать работу диспетчеров, менеджеров, складских сотрудников и курьеров. Нужны веб-интерфейс для диспетчеров и менеджеров, API-бэкенд, демонстрационные данные и базовая аналитика.

Предметная область:
- заказы на доставку;
- склады;
- транспорт/курьеры;
- маршруты;
- статусы доставки;
- складские остатки;
- KPI/дашборды;
- прогнозы и рекомендации.

Требования к прототипу:
1. Это должен быть работающий full-stack prototype: frontend + backend.
2. Функциональность должна быть реалистичной, но без избыточной сложности.
3. Нужен код, который можно запустить локально через Docker Compose.
4. Должен быть демонстрационный сценарий для показа на защите.
5. Архитектура должна быть чистой и понятной, пригодной для анализа в отчёте.
6. В решении не использовать лишние внешние зависимости, если можно обойтись стандартными и популярными.
7. Код должен быть аккуратным, модульным, с понятной структурой.
8. Сразу учитывай, что проект нужен для учебной лабораторной по вайб-кодингу: важны не только фичи, но и удобство анализа качества результата.

Предлагаемый стек:
- Backend: FastAPI + SQLAlchemy + PostgreSQL
- Frontend: React + TypeScript + Vite
- UI: Tailwind CSS + shadcn/ui
- Карта: Leaflet или MapLibre (если упростит — допускается mock map)
- Charts: Recharts
- Auth: простая demo-auth без внешних провайдеров
- Запуск: Docker Compose

Обязательные пользовательские сценарии:
1. Диспетчер видит список заказов и может назначать их на маршрут.
2. Диспетчер видит активные маршруты и статус доставок.
3. Менеджер склада видит остатки по складу и предупреждения о низких запасах.
4. Курьер видит свой маршрут и может менять статус доставки: assigned -> in_transit -> delivered / failed.
5. На dashboard отображаются KPI: число заказов, доставлено вовремя, среднее ETA, low stock alerts.
6. Должны быть демо-данные не менее чем для:
   - 3 складов
   - 30 заказов
   - 8 курьеров
   - 10 маршрутов
   - 20+ складских позиций
